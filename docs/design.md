# AI 视觉对话助手设计文档

## 1. 项目目标

本项目实现一款 Web 端 AI 视觉对话助手。用户授权摄像头和麦克风后，应用可以预览摄像头画面、识别用户语音，并让 AI 结合当前画面、最近动作、会话历史和场景模式进行中文回答，最后通过浏览器 TTS 播报。

当前方案不是上传原始连续视频流，而是采用“本地 2 FPS 采样 + 帧差去重 + WebSocket 流式关键帧 + 后端滑动视觉记忆”的近似视频流方案。浏览器持续采样并只推送有变化的小图关键帧，后端按会话维护最新画面摘要和最近动作时间线，云端模型只在节流窗口内分析有价值的帧或短序列。

## 3. 场景模式

场景模式只改变对话 prompt 和 UI 展示重点，不改变视觉摘要、动作时间线、抽帧频率或云端调用频率。

| 场景模式 | 用户故事 | 回答重点 |
| --- | --- | --- |
| 通用视觉对话 | 用户直接询问当前画面、最近动作或继续追问前文。 | 均衡参考画面摘要、动作时间线和历史对话。 |
| 动作理解助手 | 用户希望 AI 判断刚才做了什么、有没有移动、手势是否变化。 | 优先参考最近几秒动作时间线。 |
| 桌面学习助手 | 用户展示纸张、笔记、题目或桌面物品，希望 AI 辅助识别和解释。 | 优先关注画面摘要和学习/办公内容。 |
| 演讲/面试练习助手 | 用户希望 AI 观察坐姿、视线、手势和表达状态。 | 给出简短、建设性的表现反馈。 |
| 生活提醒助手 | 用户希望 AI 描述桌面物品变化和需要注意的地方。 | 偏向物品状态、变化和轻量提醒。 |

## 4. 当前架构

- `apps/web`：React + Vite 前端，负责摄像头预览、本地抽帧、帧指纹去重、WebSocket 关键帧推送、语音识别、TTS、场景模式和 UI 展示。
- `apps/server`：NestJS 后端，负责 DashScope 多模态调用、会话回复、关键帧对象存储、session 历史记忆和 `/api/omni/realtime` WebSocket 视觉记忆代理。
- `packages/shared`：前后端共享类型，包含视觉摘要、动作时间线、视频流帧、场景模式、Omni WebSocket 事件和对话请求契约。

核心流程：

1. 用户点击连接，前端申请摄像头和麦克风权限。
2. 前端启动预览、浏览器语音识别和自动观察。
3. 自动观察在本地采样关键帧，静止或低变化帧只计数不上传。
4. 有变化的关键帧被压缩后通过 WebSocket `video_frame` 事件发送给后端。
5. 后端按 session 维护滑动视觉记忆，并节流生成最新画面摘要和动作时间线。
6. 用户语音提问时，前端携带当前场景模式；WebSocket 可用时通过 `text` 事件提问，否则回退 HTTP。
7. 后端结合场景模式、视觉摘要、动作时间线和历史对话生成回复。
8. 前端展示回复并用 TTS 播报，用户可点击“停止回复”打断。

## 5. 成本控制策略

| 成本控制技巧 | 是否想到 | 实际采用 |
| --- | --- | --- |
| 不上传原始连续视频，只上传关键帧。 | 是 | 已采用。 |
| 浏览器本地 2 FPS 采样。 | 是 | 已采用，本地采样不等于云端 2 FPS 调用。 |
| 本地帧差去重。 | 是 | 已采用，低变化帧不会进入云端分析。 |
| 压缩关键帧。 | 是 | 已采用，小尺寸 JPEG 和低细节视觉分析。 |
| 默认 `detail: low`。 | 是 | 已采用，自动和手动分析默认低细节。 |
| 后端按会话节流云端调用。 | 是 | 已采用，画面摘要和动作时间线分别设置分析间隔。 |
| 后端限制最大帧大小。 | 是 | 已采用，超大 WebSocket 关键帧会跳过。 |
| 页面不可见、AI 播报、后端离线时暂停采样/上传。 | 是 | 已采用。 |
| WebSocket 失败时 HTTP fallback。 | 是 | 已采用，保证演示稳定。 |
| 场景模式不增加云端调用。 | 是 | 已采用，只改变 prompt 和 UI 展示重点。 |
| 用户级预算、调用审计和数据库报表。 | 是 | 暂未实现，作为后续增强。 |

## 6. 关键接口

- `POST /api/vision/analyze`：单帧画面分析，用于手动分析和视觉问题兜底。
- `POST /api/vision/analyze-sequence`：HTTP 多帧动作分析，作为 WebSocket 不可用时的兜底。
- `POST /api/conversation/respond`：HTTP 对话回复兜底，携带 `sceneMode`、视觉摘要、动作时间线和历史上下文。
- `WS /api/omni/realtime`：流式视觉记忆与对话通道。
  - 客户端事件：`start`、`video_frame`、`vision_context`、`text`、`stop`。
  - 服务端事件：`ready`、`video_summary`、`action_timeline`、`video_status`、`text`、`error`、`closed`。

## 7. 后续方向

1. 接入 DashScope 原生实时音视频能力，替换当前浏览器语音识别和关键帧近似方案。
2. 增加服务端用户级限流、预算告警、调用审计和成本报表。
3. 将会话、消息、视觉摘要、动作时间线和指标持久化到数据库。
4. 增加端到端演示脚本，覆盖摄像头授权、WebSocket 视频流、语音提问、动作问题和降级路径。
5. 引入更强的动作理解策略，例如更长窗口、多尺度关键帧选择或轻量本地姿态估计。
